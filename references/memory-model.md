# Memory Model

This skill keeps two lightweight memory layers inspired by EvoScientist.

## Ideation Memory

Use this after problem framing and idea ranking.

```md
## Ideation Memory

- Objective:
- Promising directions:
- Rejected directions:
- Why rejected:
- Novelty cues:
- Evaluation criteria:
- Open questions:
```

## Experimentation Memory

Use this after implementation, debugging, or analysis.

```md
## Experimentation Memory

- Experiment or task:
- Pilot scope:
- Successful tactics:
- Failed tactics:
- Failure modes:
- Best observed result:
- Reproducibility notes:
- Next recommended step:
```

## Rules

- Keep memories terse and reusable.
- Record failures with reasons, not just outcomes.
- Do not store speculation as established fact.
- Update memory at the end of each meaningful iteration, not only at the end of the project.
